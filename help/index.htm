KwikStrip Documentation

Thanks for grabbing a copy of KwikStrip! This help page should let you know how everything works. I'll try not to get too programmatic, but that's the way I think. Anyway, there is a topic list over there, you can use it to jump around. Have fun using KwikStrip, I hope it serves as an excellent application management solution.

This document covers Version 2.5 the information might be inaccurate or completely void if you are using an older version. You can upgrade here at the KwikStrip page.

Oh, I should also point out that any icons shown on the strip in any of the screenshots on this page (or anywhere for that matter) are the property of their respective owners, but you knew that already right?

What is KwikStrip?

KwikStrip is an application launcher, but more specifically, it's this:

Okay, that doesn't look like much. That's the point. KwikStrip is designed to take as little amount of space as possible until you tell it to spring to life by clicking on it. Then it opens up to reveal your buttons. Like this:

Why should I use it?

Well, there are several reasons why you should at least try KwikStrip:

  • Navigating through the Start Menu is a pain.
  • The Start Menu is especially evil if you have installed a zillion programs and the Start Menu is full of a zillion submenus and in most cases you aren't even sure where all those submenus came from...
  • Sometimes it's impossible to organize the Start Menu due to user account restrictions.
  • Using the built-in QuickLaunch bar wastes valuable taskbar space. Especially when you need to add 20 or 30 items. KwikStrip solves this problem. I've personally seen strips as large as 140 buttons!
  • Forget searching the Start Menu, launching an application with KwikStrip is simple as Click, Click, Done!

How to report a bug

KwikStrip has been in development for a long time. I've worked out nearly all the bugs. Version 2.5.6 is very stable. Nevertheless, I'm not perfect so the chances of my program being perfect are pretty slim. If you do happen to find a bug, there's a bug tracker on my web site so go check that out. If the problem isn't already listed, you can report it using the handy bug report area of my forum.

If you are reporting a crash, make sure to post everything that comes out of the crash dialog and your config/ini file. Also make sure you describe what you were doing when the bad stuff happened. Nothing is worse than reporting a big problem where I can't figure out how you got it.

Getting Started

Kwikstrip isn't difficult to install and use. I've tried to keep it very simple. For the sake of completeness, here's installation and basic usage instructions.

Installing KwikStrip

Installing KwikStrip depends on how fresh you want it. In most cases, you will want to grab the latest official version from my site on the KwikStrip page. If you want it hot out of the oven, you will need to check-out a copy from the SVN server.

Once you have a .zip, .tar.bz2 or even just the .exe, Just drop it in a folder somewhere and run it. No installation, no funky registry stuff, now weirdo wizards.

If you want KwikStrip to start when you turn on your computer, make a shortcut in your start menu's "startup" folder. You could do a registry thing too, but KwikStrip will never ever touch the registry so you'll have to do it yourself. It's pretty easy, just fire up RegEdit and browse to: 

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run

Then add a key named "KwikStrip" and set the value to wherever you put KwikStrip.exe.

I prefer the startup folder method since I tend to move KwikStrip around a lot and startup is easy to update.

Adding an item

The first time you start KwikStrip, it will ask you to configure it, click "yes" to enter the configuration dialog. Ignore all the pretty knobs and switches. In the lower-left corner there is a button labeled "add", you want that. Hit it now to enter the item properties dialog. Type in the name of the program under "Application Name" and click the "..." button next to Executable/File/Directory/URL path and find the .exe using the file browser thing. Ignore everything else, We'll talk about that stuff later in the Optional item features section. Click "OK" when you're finished.

Now that you have an item (conveniently shown in the item list), you can run KwikStrip, but why would you want a strip with only 1 item? Take some time to add more items and fill out your strip. When you're done adding items, click "OK" to save your configuration and start KwikStrip for real.

The main window

The main window is quite simple, containing only 3 parts:

  1. The Arrow: The arrow is what opens and closes the strip. Without it, KwikStrip would use up a ton of screen space all the time and nobody wants that now do they?
  2. Buttons: KwikStrip's buttons are what launch the applications you define in the configuration. Just a single click and away you go.
    Right-clicking on a button will bring up a special menu with extra launching options. These options are covered in the Extra Launching Features section.
  3. The "InfoBox": The InfoBox displays the item title of the button your mouse pointer is hovering over or just the version of KwikStrip is running.
    Right-clicking on a button will bring up the system menu which will bring up the configuration or on-line help.

The skin file format

Yeah, there will be some info here, but I'm just keeping the skin specs for now so I don't lose them. 

typedef struct{
    // BITMAP bmp; This is just the raw bitmap data.
    // The skin is basically just tacked onto the end of a standard windows bitmap file.
    // You shouldn't need to read anything from it.
    char format[9];        // This should always be read as "KWIKSTRIP"
    unsigned char version; // Version ID, this is version 0x03
    unsigned char namelen; // how many chars long .name is (not including \0)
    char name[->namelen];  // the title of the skin
    unsigned char fontlen; // how many chars long .font is (not including \0)
    char font[->fontlen];  // the font name to be used for the infobox
    unsigned char mxlen;   // how many bytes to read for .mx
    char mx[->mxlen];      // the skin metrics data
    signed short offset;   // Number of bytes from the end of the file this skin def is
}SKINDEF;

And the metrics struct:

typedef struct{
    unsigned long txc; // Color of the infobox font, stored little-endian (#BBGGRR)
    unsigned char txt; // Size of the infobox text (in pixels I think, not sure)
    unsigned char txa; // Bit 0x1 means bold, 0x2 means italic. (you can also have both)
    unsigned char arw; // X pos of the arrow graphic on skn->bmp
    unsigned char btn; // X pos of the button graphic on skn->bmp
    unsigned char ilf; // X pos of the left side of the infobox graphic on skn->bmp
    unsigned char ict; // X pos of the center of the infobox graphic on skn->bmp
    unsigned char irt; // X pos of the right side of the infobox graphic on skn->bmp
    unsigned char up;  // Y pos of the up states on skn->bmp
    unsigned char hv;  // Y pos of the hover states on skn->bmp
    unsigned char dn;  // Y pos of the down states on skn->bmp
    unsigned char inf; // Y pos of the infobox parts on skn->bmp
    unsigned char arx; // Width of the arrow graphic
    unsigned char ary; // Height of the arrow graphic
    unsigned char bnx; // Width of the button graphic
    unsigned char bny; // Height of the button graphic
    unsigned char inx; // Width of the infobox sections
    unsigned char iny; // Height of the infobox sections
    unsigned char icx; // Width to draw icons on the strip
    unsigned char icy; // Height to draw icons on the strip
    unsigned char bpx; // Amount of space between the left edge of a button and it's icon
    unsigned char bpy; // Same as ->bpx except this is for the top
    unsigned char bdx; // Same as ->bpx except this is when the button is pressed
    unsigned char bdy; // Same as ->bpy except this is when the button is pressed
    unsigned char ipx; // Amount of space between the left edge of the infobox and the text
    unsigned char ipy; // Same as ->ipx except this is for the top
}METRICS={0xDDDDDD,12,1,0,18,0,18,36,0,36,72,108,18,36,36,36,18,18,32,32,2,2,3,3,2,2};
